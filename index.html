SNAP Retailers Map (Washington, D.C.)

An interactive web map built with Mapbox GL JS, Turf.js, and custom HTML/CSS/JavaScript, showing Supplemental Nutrition Assistance Program (SNAP) retailers across Washington, D.C. Using data from the USDA SNAP Retailer Locator, I designed the map so users can explore food access visually through filters for store type and ward boundaries, dynamic legends, and color-coded markers. Each store is represented by a unique symbol and color, and clicking reveals detailed information with options to copy the address or generate walking directions via Google Maps. I implemented clustering to manage dense data points, a DC boundary mask for geographic focus, and ward selection both from dropdown menus and interactive map clicks.

The "Near Me" panel integrates geolocation, search, and map-pick functionality to return the nearest retailer by type with real-time distance calculations. I also built a responsive filters panel, an onboarding modal, and accessibility-minded popups styled with custom CSS — combining spatial data, mapping libraries, and interface design into something that's actually easy to use.

Wildfire Burn Severity Mapping (Clear Fire, Alaska – 2024)

Multispectral Sentinel-2 satellite imagery and a Random Forest classifier, used to map burn severity following the 2024 Clear Fire in Alaska. I built a full remote-sensing workflow that preprocesses imagery in Google Earth Engine, extracts vegetation and moisture indices, and applies a machine-learning model in Python to classify fire impacts at 10-meter resolution.

The model achieved near-perfect accuracy when reproducing USGS burn-severity classes based on the dNBR index, confirming correct data handling, spatial alignment, and classification logic. The project demonstrates skills in geospatial data processing, raster analysis, supervised classification, and cartographic design using tools such as Rasterio, Scikit-learn, QGIS, and GDAL.

Health Equity Index of Seattle

This project maps health equity in Seattle using a composite index built from five metrics:

  • Adult diabetes rates
  • Adult asthma rates
  • Life expectancy at birth
  • Adult disability rates
  • Health insurance coverage

I analyzed 2017–2021 data by census tract in ArcGIS Pro to reveal disparities. The analysis shows that neighborhoods such as Rainier Valley and southern Delridge face poorer health outcomes, while areas like Montlake and Ballard perform better.

The project shows how GIS can guide public health planning by showing where resources are most needed. These maps can support non-profits and government agencies in targeting interventions in underserved communities.

D.C. Crime

An exploration of crime patterns in Washington, D.C. using publicly available data from the city's Open Data Portal. I cleaned and processed incident-level records, then built an interactive web map in ArcGIS Online that lets users zoom into neighborhoods, filter by category, and view spatial patterns at both the citywide and block level.

To complement the interactive map, I also produced two cartographic layouts in ArcGIS Pro: a proportional symbol map, where circle sizes represent the number of incidents by location, and a choropleth map that aggregates incidents by neighborhood boundaries for a normalized view of crime density. Between the two, you get both an at-a-glance overview and a way to dig into specifics.

Poverty in King County

This project analyzes poverty in King County and measures data reliability using the CDC Social Vulnerability Index ACS Toolbox.

Maps created

  1. Population Below 150% of Poverty Level: Choropleth map showing estimated percentages by census tract.
  2. Coefficient of Variation for Poverty Estimates: Diverging scheme that highlights low, moderate, and high variability.

Process

  • Used ACS data through the CDC SVI dataset
  • Calculated poverty percentages in ArcGIS Pro
  • Measured reliability with the CV Calculator

Key insights

  • Higher concentrations of poverty appear in urban areas near Seattle
  • Some tracts show higher variability, so users should be careful with interpretation

This work links quantitative analysis with clear mapping. It shows not just where poverty is concentrated, but also how reliable the estimates are.

Digital Terrain Model

I processed pre- and post-flood LiDAR datasets to evaluate changes in canopy height and terrain elevation, applying raster differencing to identify areas most impacted by flooding. The workflow involved generating canopy height models (CHMs) and digital terrain models (DTMs), validating coordinate systems and resolutions, and ensuring data quality by checking for null values. I then created difference rasters, plotted histograms to guide classification, and visualized spatial patterns of canopy loss, terrain shifts, and potential flood damage.

I used tools like xarray and rasterio to process the multi-dimensional LiDAR data, built classified raster maps with custom colormaps, and used matplotlib for the final plots — turning raw LiDAR into a clear read on where canopy and terrain changed most.

Kent Fire Department Incident Analysis

To better understand patterns in emergency service demand, I analyzed Kent Fire Department's 2016 incident records using GIS. The workflow began by cleaning and geocoding the raw call data to ensure accurate spatial representation of incidents across the service boundary. Once validated, I separated incidents into meaningful categories such as structure fires, hazmat, rescue/EMS, and service calls. This allowed for focused analysis on different operational demands. Kernel density estimation and contour mapping were then applied to reveal spatial concentrations of responses, providing clear visualization of where resources were most heavily utilized.

In addition to density mapping, I created category-specific heatmaps to compare distributions across emergency types. For example, Rescue/EMS responses showed broad, high-density coverage across residential areas, while structure fire incidents clustered more tightly around central Kent and Covington. Contour intervals, symbology, and grid values were layered on top to show incident density per square mile alongside the broader spatial patterns.
