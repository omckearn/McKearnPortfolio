SNAP Retailers Map (Washington, D.C.)

This project is an interactive web map built with Mapbox GL JS, Turf.js, and custom HTML/CSS/JavaScript, showcasing Supplemental Nutrition Assistance Program (SNAP) retailers across Washington, D.C. Using data from the USDA SNAP Retailer Locator, I designed the map to allow users to explore food access visually through filters for store type and ward boundaries, dynamic legends, and color-coded markers. Each store is represented by a unique symbol and color, and clicking reveals detailed information with options to copy the address or generate walking directions via Google Maps. I implemented clustering to manage dense data points, a DC boundary mask for geographic focus, and ward selection both from dropdown menus and interactive map clicks.

Beyond visualization, I developed features that highlight my front-end and geospatial problem-solving skills. The “Near Me” panel integrates geolocation, search, and map-pick functionality to return the nearest retailer by type with real-time distance calculations, demonstrating proficiency with geospatial analysis in the browser. I also built user-friendly UI elements such as a responsive filters panel, onboarding modal, and accessibility-minded popups styled with custom CSS. This project exemplifies my ability to combine spatial data acquisition, interactive mapping libraries, and thoughtful design to create tools that communicate complex datasets in an intuitive, impactful way.

Health Equity Index of Seattle

This project maps health equity in Seattle using a composite index of five metrics:

  • Adult diabetes rates
  • Adult asthma rates
  • Life expectancy at birth
  • Adult disability rates
  • Health insurance coverage

I analyzed 2017–2021 data by census tract in ArcGIS Pro to reveal disparities. The analysis shows that neighborhoods such as Rainier Valley and southern Delridge face poorer health outcomes, while areas like Montlake and Ballard perform better.

The project shows how GIS can guide public health planning by showing where resources are most needed. These maps can support non-profits and government agencies in targeting interventions in underserved communities.

D.C. Crime

This project explores patterns of crime in Washington, D.C. using publicly available data from the city’s Open Data Portal. I cleaned and processed incident-level records, then built an interactive web map in ArcGIS Online that allows users to zoom into neighborhoods, filter by category, and view spatial patterns at both the citywide and block level. The web map demonstrates how open data and web GIS can provide accessible insight into public safety trends.

To complement the interactive map, I also produced two cartographic layouts in ArcGIS Pro. The first is a proportional symbol map, where circle sizes represent the number of incidents by location, making it easy to compare concentrations across the city. The second is a choropleth map that aggregates incidents by neighborhood boundaries, providing a normalized view of crime density and distribution. Together, these products highlight my ability to combine web GIS tools with traditional cartography, ensuring that the analysis is both interactive and publication-ready.

This project demonstrates skills in data acquisition, cleaning, and visualization across multiple GIS platforms. By using both ArcGIS Online and ArcGIS Pro, I show how spatial data can be tailored to different audiences: decision-makers who need high-level summaries and community members who benefit from an interactive way to explore crime data.

Poverty in King County

This project analyzes poverty in King County and measures data reliability using the CDC Social Vulnerability Index ACS Toolbox.

Maps created

  1. Population Below 150% of Poverty Level: Choropleth map showing estimated percentages by census tract.
  2. Coefficient of Variation for Poverty Estimates: Diverging scheme that highlights low, moderate, and high variability.

Process

  • Used ACS data through the CDC SVI dataset
  • Calculated poverty percentages in ArcGIS Pro
  • Measured reliability with the CV Calculator

Key insights

  • Higher concentrations of poverty appear in urban areas near Seattle
  • Some tracts show higher variability, so users should be careful with interpretation

This work links quantitative analysis with clear mapping. It shows not just where poverty is concentrated, but also how reliable the estimates are.

Digital Terrain Model

I processed pre- and post-flood LiDAR datasets to evaluate changes in canopy height and terrain elevation, applying raster differencing to identify areas most impacted by flooding. The workflow involved generating canopy height models (CHMs) and digital terrain models (DTMs), validating coordinate systems and resolutions, and ensuring data quality by checking for null values. I then created difference rasters, plotted histograms to guide classification, and visualized spatial patterns of canopy loss, terrain shifts, and potential flood damage.

This project demonstrates skills in remote sensing, raster analysis, and geospatial data visualization using Python. I applied tools such as xarray and rasterio to process multi-dimensional LiDAR data, built classified raster maps with custom colormaps, and used matplotlib for clear, professional plots. The analysis highlights my ability to translate raw LiDAR data into actionable environmental insights, while showcasing strong technical proficiency in GIS workflows, data quality assurance, and map production.

Kent Fire Department Incident Analysis

To better understand patterns in emergency service demand, I analyzed Kent Fire Department’s 2016 incident records using GIS. The workflow began by cleaning and geocoding the raw call data to ensure accurate spatial representation of incidents across the service boundary. Once validated, I separated incidents into meaningful categories such as structure fires, hazmat, rescue/EMS, and service calls. This allowed for focused analysis on different operational demands. Kernel density estimation and contour mapping were then applied to reveal spatial concentrations of responses, providing clear visualization of where resources were most heavily utilized.

In addition to density mapping, I created category-specific heatmaps to compare distributions across emergency types. For example, Rescue/EMS responses showed broad, high-density coverage across residential areas, while structure fire incidents clustered more tightly around central Kent and Covington. By integrating cartographic techniques such as contour intervals, symbology, and grid values, the maps illustrate patterns while also quantifying incident density per square mile. This project highlights my ability to combine data processing, spatial analysis, and effective visualization to support decision making in public safety resource allocation.
